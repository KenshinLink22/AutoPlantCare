Auto Plant Care

A Standalone Plant Monitoring & Balancing System

By: Nathan Leung & Angel Hernandez Vega

EEC 172 - Embedded Systems

1. Project Description

The Auto Plant Care system is a standalone plant monitoring and balancing control system designed for plant owners who may be away from home for extended periods. It actively monitors basic factors of plant care, such as the volumetric water content (VWC) of the soil and ambient light intensity.

Unlike simple passive monitors, this system physically interacts with the environment. It utilizes a motorized dual-film polaroid screen to dynamically regulate light exposure, and an automated water pump to maintain soil moisture. The user can interact with the system locally using an IR remote to set the desired moisture threshold, which is displayed on an OLED screen. Furthermore, the system is connected to the cloud via AWS IoT and SNS, providing real-time email push notifications to the user whenever the plant is successfully watered.

3. Design

Functional Specification

The system operates in a continuous polling loop, maintaining both moisture and light levels:

  • Moisture Control: The system reads I2C data from a soil moisture sensor. If the actual moisture falls below the target moisture (configured dynamically via IR remote), a GPIO pin triggers a relay to power a DC water pump. Once the target is reached, the pump shuts off and an HTTP POST request is sent via TLS to an AWS API endpoint, triggering an SNS email alert to the user.
  • Light Control: Two photoresistors read ambient light levels via the CC3200's ADC channels. If the light exceeds the maximum threshold, a DC motor adjusts the position of a polarized screen to block the strongest light source, utilizing end-stops for positional feedback.

System Architecture

Figure 1: System Architecture Diagram

The TI CC3200-LAUNCHXL serves as the core microcontroller. The system features the following subsystems:

  • Inputs: I2C SparkFun Soil Moisture Sensor, 2x ADC Photoresistors, GPIO IR Receiver, 2x GPIO End-stop switches.
  • Outputs: SPI Adafruit SSD1351 OLED Display, GPIO Relay (Water Pump), GPIO Relay (DC Motor).
  • Cloud: 802.11 b/g/n Wi-Fi connection to AWS IoT Core (REST API) routed to AWS Simple Notification Service (SNS).

5. Challenges

During the development of the Auto Plant Care system, several significant challenges were encountered and resolved:

  • Hardware Pin Conflicts: During the integration phase, the SPI clock for the OLED display conflicted with the GPIO input needed for the IR sensor. Because the default SPI CLK (Pin 5) was damaged on the board, it was re-routed to Pin 45 via pin multiplexing. However, Pin 45 was later needed for the IR receiver. This was resolved by meticulously mapping the IR receiver to an isolated GPIO (Pin 18) and utilizing the CC3200 SysConfig tool to ensure peripherals did not overlap.
  • AWS Integration & JSON Parsing: The initial AWS IoT integration resulted in successful HTTP 200 OK responses, but the SNS emails were not triggering. It was discovered that the AWS IoT Rule SQL query (SELECT state.desired.default) mismatched the JSON key generated by the C code (state.desired.var). Ensuring strict key-value parity across the embedded C payload and the cloud SQL query resolved the silent failures.

6. Future Work

If given more time, several improvements could be made to the system:

  • Implementing a water-level sensor in the reservoir tank to send an AWS email alert when the pump is running out of water.
  • Creating a custom web-dashboard using AWS Amplify to log historical moisture and light data over time, rather than just sending email alerts.
  • Refining the mechanical housing using 3D printed parts for a more robust polaroid screen rotation mechanism.
